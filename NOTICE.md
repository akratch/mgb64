## SDK and third-party material

The tree currently contains inventoried Nintendo 64 SDK/libultra-lineage
compatibility material for the in-progress matching target. Tracked proprietary
SDK notice text has been removed or replaced, but not all compatibility material
is original MGB64 work or offered under the MGB64 MIT License. This remains a
cleanup priority.

The optional N64 matching target has a public placeholder at `src/bootcode.s`.
It does not contain IPL3 boot ROM code or boot-font data. Matching-target work
that needs boot material must supply it locally from a legally obtained source;
those local files or generated bytes must not be committed or redistributed.

Third-party tools and libraries are listed in `THIRD_PARTY.md` with their
origins and licenses.

## SMAA antialiasing lookup tables

The optional SMAA (Enhanced Subpixel Morphological Antialiasing) filter uses two
precomputed lookup-table textures generated from the SMAA reference
implementation by Jorge Jimenez et al. (<https://github.com/iryoku/smaa>,
commit `71c806a`). The reference generator scripts are vendored verbatim under
`tools/smaa/` and the wrapper `tools/smaa/gen_luts.py` emits the committed C
headers `src/platform/fast3d/smaa_area_tex.h` and `smaa_search_tex.h`. No
ROM-derived data is involved. The SMAA reference implementation is provided
