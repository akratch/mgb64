## FID-0064 — `generate_ammo_total_microcode` omits the retail `mpmenuon` early-out

**Function:** `generate_ammo_total_microcode` (US), ASM `glabel` at `src/game/gun.c:32343`.
Live C body: `src/game/gun.c:32053-32069` (`#ifdef NATIVE_PORT`).

**Retail prologue (ASM, gun.c:32351-32356):** two independent `!= 0` early-outs
before any HUD geometry is built:

```
lw    $t6, 0x1064($v0)   ; g_CurrentPlayer->gunammooff
bnezl $t6, <return>
lw    $t7, 0x29c4($v0)   ; g_CurrentPlayer->mpmenuon
bnezl $t7, <return>
```

Struct offsets confirmed against `src/game/bondview.h`: `gunammooff` (line 1043)
= `0x1064`, `mpmenuon` (line 2272) = `0x29c4`.

**Divergence:** the live C checks only the first guard —
`if (g_CurrentPlayer->gunammooff != 0) return gdl;` (gun.c:32056) — and its
delegate `portDrawHandAmmo` (gun.c:31959) has no `mpmenuon` guard either. The
retail `mpmenuon` suppression is dropped.

**Reachability (confirmed):** the caller (`bondview.c:21384`) invokes
`generate_ammo_total_microcode` unconditionally. The caller's own `mpmenuon`
block (`bondview.c:21242-21251`) only *appends* the menu-darkening overlay
(`microcode_constructor` + `microcode_constructor_related_to_menus`) and does
**not** early-return, so the internal guard is the sole gate. `mpmenuon` is set
`TRUE` by `mp_watch.c:353/642/708` when the MP watch/pause menu opens.

**Effect:** in multiplayer with the watch menu open, retail suppresses the entire
per-pane ammo HUD; the port still draws the ammo icon + clip count + reserve for
both hands over/behind the darkening overlay. Visual only — no sim/determinism
impact. The same `mpmenuon` HUD-suppress pattern *is* honored elsewhere in the
port (`radar.c:63`, `minimap.c:828`, `bondview.c:22840/22884`) but was missed
here. Class: port-defect, surface renderer (authority = retail ASM, which this
function has).

## FID-0065 — `handles_projectile_motion` endpoint pull-back: inverted guard + wrong base operand

**Function:** `handles_projectile_motion`, ASM `src/game/chrobjhandler.c:5396-5932`.
Live C body: `src/game/chrobjhandler.c:5033-...` (`#ifdef NONMATCHING`).

The whole function verifies equivalent **except** the after-room-loop endpoint
pull-back block. `result` (stack slot `0x2a4`) is initialised `1` (C:5069) and set
`0` only when a background/wall hit is recorded (`sw $zero, 0x2a4` ~ASM:5615;
C:5186 `result = 0`). `dest` is initialised to `arg1` (C:5079-5081).

**Divergence 1 — inverted guard.** ASM (chrobjhandler.c:5660):

```
lw    $t5, 0x2a4($sp)          ; $t5 = result
bnezl $t5, .L7F0426FC          ; branch-likely: if result != 0, SKIP the block
```

so the block runs **only when `result == 0`** (a wall hit occurred). The live C
runs the block on the *opposite* branch: `if (result != 0) { ... }` (C:5207).

**Divergence 2 — wrong base operand.** ASM reads the base from `arg2` (`$s3`):

```
lwc1 $f6, ($s3)                ; base = arg2.x  (chrobjhandler.c:5698)
sub.s $f16, $f6, $f8           ; arg2.x - frac*delta.x
swc1 $f16, ($s3)               ; store back to arg2
... copy arg2 -> dest(sp+0x268) ; chrobjhandler.c:5712-5716
```

`$s3` is `arg2` (call site passes it as the 5th arg: `sw $s3, 0x10($sp)`,
chrobjhandler.c:5724). The live C reads the base from `dest` (=`arg1`):
`arg2->x = dest.x - frac * delta.x; ... dest.x = arg2->x;` (C:5220-5226).

`frac = 0.1/dist` (or `0.5` when `dist <= 0.1`); `delta = saved_arg1 -
runtime_pos` (identical in both).

**Net (values passed to `projectileFindCollidingProp`), enumerated:**

| case | retail ASM (dest, arg2) | live C (dest, arg2) |
|------|-------------------------|---------------------|
| wall hit (`result==0`) | `bg_hit - 0.1*travel`, `bg_hit - 0.1*travel` | `arg1` (through wall), raw `bg_hit_pos` |
| no hit (`result!=0`)   | `arg1`, (caller buffer) | `arg1 - 0.1*travel`, `arg1 - 0.1*travel` |

Neither case reconciles → this is a genuine mis-transcription, not a compensating
decompiler restructuring. Retail *clamps* the projectile's collision-test
endpoint and reported impact to just before the wall on a hit; the port instead
skips that clamp on hit and applies the `-0.1` shorten only on the no-hit path.

**Reachability (path confirmed, type-set not enumerated):** the deposit-motion
path — `sub_GAME_7F042EB4` (chrobjhandler.c:6403) calls this under
`(runtime_bitflags & RUNTIMEBITFLAG_DEPOSIT) && projectile->flags & 4`, dispatched
from the object-physics tick (chrobjhandler.c:9452/9562/9946). The divergence is
unconditional *within* the function. **Not yet verified:** exactly which
projectile/weapon types set `projectile->flags & 4`, and runtime pixel/trace
confirmation of the through-wall impact (this derivation is static). Class:
port-defect, surface sim.
