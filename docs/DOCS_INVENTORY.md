# Documentation Inventory & Currency Triage

A maintainer-facing triage of every tracked Markdown doc: is it current, does it
need an update, is it obsolete (overcome-by-events), or is it an owner judgment
call? This is a point-in-time curatorial pass (2026-07-11) done against the
**actual code/state** — each verdict cites concrete evidence (a symbol that
exists or doesn't, a guard/`.gitattributes` reference, a checkbox/ledger state,
or a git fact), not a vibe.

**Scope.** All tracked `*.md` **except** `docs/design/remaster-aaa/**` (owned by a
concurrent remaster effort — see the flag at the bottom). Verdicts:

- **CURRENT** — accurate to reality; leave as-is.
- **UPDATE (done)** — had a stale claim; fixed this pass.
- **OWNER-DECIDE** — a judgment call deferred to the maintainer (removal vs keep,
  or a re-attestation that is engineering work, not a doc edit).
- No doc met the bar for **OBE-REMOVE** this pass: every removal candidate turned
  out to be referenced by an active plan, load-bearing in a release guard, or a
  live forward-looking plan. Nothing was deleted. See "Removal candidates" below.

Method: `tools/check_markdown_links.py` (link/anchor integrity), a phantom
tool/script path sweep across all in-scope docs, cross-reference against
`docs/BACKLOG_v0.4.0.md`, `docs/fidelity/LEDGER.md` / `ASM_AUDIT.md`, and direct
`grep` of `src/`, `scripts/`, `tools/`, and `.gitattributes`.
